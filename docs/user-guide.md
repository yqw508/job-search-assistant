# 用户指南

## 推荐使用流程

1. 启动本地服务：运行 `start_service.bat`。
2. 打开 Boss，手动浏览岗位。
3. 对感兴趣的岗位点击扩展里的“收藏当前岗位”。
4. 在本地后台查看匹配度和职位描述。
5. 对值得推进的岗位切换状态，例如“已投递”“面试中”。
6. 面试后在岗位详情或“面试记录”页记录问题和表现。
7. 在技能库查看高频技能和薄弱技能。
8. 在项目经验里补充能证明这些技能的项目材料。

## 岗位收集

扩展适合在两种 Boss 页面使用：

- 左侧列表、右侧详情的搜索页。
- 独立岗位详情页。

如果发现公司名、职位描述或规模字段不完整，优先打开岗位详情页后再收藏一次。相同岗位会按标准化后的 Boss 详情 URL 合并，不会重复入库。

## 技能库

技能点来源：

- 职位描述自动提取。
- 面试问题手动关联。
- 项目经验手动关联。

技能库不是简历，它更像个人能力台账：记录哪些技能被岗位反复要求、哪些技能在面试里暴露短板、哪些项目可以支撑这些技能。

## 面试记录

每次面试建议记录：

- 面试轮次。
- 面试形式。
- 面试结果。
- 整体表现分。
- 面试问题。
- 每个问题关联的技能点。
- 每个问题的回答表现分和复盘。

后台会根据问题关联的技能点统计：

- 被问最多的技能。
- 低分次数最多的技能。
- 平均表现分。

## 数据备份

重要数据都在 SQLite 文件中：

```text
output/boss_jobs.sqlite3
```

备份这个文件即可迁移数据。
