# 扩展手工验证清单

## 1. 启动服务

在项目目录执行：

```powershell
.\start_service.bat
```

看到本地服务启动后，访问：

```text
http://127.0.0.1:8765/health
```

返回 `ok=true` 表示服务正常。

## 2. 加载扩展

1. 打开 Chrome 或 Edge 扩展管理页。
2. 开启开发者模式。
3. 点击“加载已解压的扩展”。
4. 选择项目的 `extension` 目录。

## 3. 收藏岗位

1. 用正常浏览器打开 Boss。
2. 搜索岗位。
3. 手动打开一个岗位详情。
4. 点击扩展图标。
5. 点击“收藏当前岗位”。
6. 预期看到“已收藏”和匹配度分数。

## 4. 查看后台

打开：

```text
http://127.0.0.1:8765
```

预期能看到：

- 总览页有岗位统计。
- 岗位列表出现刚收藏的岗位。
- 岗位详情页有职位描述和匹配分析。
- 岗位详情页可以新增面试记录。

## 5. 取消收藏

1. 在 Boss 再次打开该岗位详情。
2. 点击扩展里的“取消收藏当前岗位”。
3. 刷新本地岗位列表。
4. 预期该岗位不再出现在默认列表中。

## 6. 安全边界

扩展不应该自动点击岗位卡片，不应该循环打开详情页，也不应该控制浏览器跳页。它只读取当前页面已展示的岗位内容。
