# Boss 岗位助手设计文档

## 目标

构建一个本地半自动浏览器助手，用于帮助用户从 Boss 直聘中采集、筛选和排序适合自己的公司岗位，辅助个人求职决策。

程序会打开一个可见浏览器，由用户手动登录 Boss 账号。登录完成后，程序按照配置搜索广州 Java/Spring Boot 岗位，以低频方式读取岗位列表和部分详情页，根据用户画像过滤不合适岗位，对候选岗位打分，并导出 Excel 表格。

## 用户画像

- 技术栈：Java、Spring Boot。
- 工作地：广州。
- 薪资范围：22K 及以上。
- 公司规模：100 人及以上。
- 排除项：外包、驻场外包、派遣、短期外包项目。
- 偏好：C 端产品和 C 端业务岗位。

## 安全与合规边界

- 程序不保存 Boss 账号和密码。
- 程序不绕过验证码、身份验证、访问控制或平台限流。
- 用户在可见浏览器中手动登录。
- 采集行为保持低频，并通过配置限制最大采集页数。
- 工具只整理用户登录后正常可见的岗位信息。

## 推荐方案

使用 Python、Playwright 和 openpyxl 实现。

Playwright 负责控制可见 Chromium 浏览器。用户扫码或手动完成登录后，脚本进入搜索页，提取岗位卡片信息，按需打开详情页，应用过滤和评分规则，最后导出 Excel。

相比直接请求接口，这种方式不需要处理账号密码，维护成本更低，也更容易在页面结构变化时通过调整选择器来修复。

## 模块设计

### `config.yaml`

保存用户偏好和运行参数：

- 搜索关键词。
- 城市。
- 最低薪资。
- 最低公司规模。
- 最大采集页数。
- 页面操作延迟范围。
- 加分关键词和排除关键词。
- 输出目录。

### `boss_job_assistant.py`

主入口：

- 加载配置。
- 启动可见浏览器。
- 等待用户手动登录。
- 执行岗位搜索。
- 遍历搜索结果页。
- 调用解析、评分和导出模块。
- 输出简洁的运行摘要。

### `boss_parser.py`

负责页面解析：

- 读取搜索结果页中的岗位卡片。
- 在需要时打开岗位详情页。
- 提取岗位名称、薪资、地点、经验、学历、公司名称、行业、融资阶段、公司规模、岗位链接和岗位描述。
- 对缺失字段做容错处理，尽量保留已采集到的信息。

### `scorer.py`

负责过滤和评分：

- 硬性过滤：广州、薪资下限不低于 22K、公司规模不低于 100 人、非外包。
- 技术加分：Java、Spring Boot、Spring Cloud、微服务、Redis、MySQL、MQ、高并发。
- C 端加分：C 端、App、小程序、电商、交易、支付、会员、用户增长、内容、社区。
- 负面信号：外包、驻场、派遣、银行外包、项目外包、短期项目。
- 输出匹配分、命中原因和排除原因。

### `exporter.py`

负责导出 Excel 文件到 `output/` 目录：

- 每个岗位一行。
- 字段包括：匹配状态、匹配分、岗位名称、薪资、地点、经验、学历、公司、行业、融资阶段、公司规模、岗位链接、命中原因、排除原因、岗位描述摘要。
- 设置可读的列宽和基础筛选。

## 数据流程

1. 用户运行脚本。
2. 程序打开浏览器。
3. 用户手动登录 Boss。
4. 脚本按照配置的关键词和城市搜索岗位。
5. 脚本读取搜索结果页，最多采集 `max_pages` 页。
6. 脚本解析岗位卡片和可访问的详情页。
7. 评分模块过滤并排序岗位。
8. 导出模块写入 Excel 文件。
9. 用户查看表格，决定后续投递目标。

## 异常处理

- 如果未检测到登录状态，脚本持续等待并提示用户完成登录。
- 如果页面选择器失效，解析模块记录部分字段并继续运行。
- 如果详情页加载失败，脚本保留列表页数据，并标记详情缺失。
- 如果出现平台验证页面，脚本暂停并提示用户手动处理。
- 如果没有采集到岗位，脚本不生成空 Excel，并打印可能原因。

## 测试策略

- 为薪资解析、公司规模解析、关键词匹配、外包识别和评分计算编写单元测试。
- 使用固定岗位样例测试 Excel 导出。
- 在有保存页面快照时，为解析模块增加 HTML 快照冒烟测试。
- 使用较小的 `max_pages`，例如 1，做可见浏览器端到端手动测试。

## 第一版范围

第一版实现：

- 配置驱动的岗位画像。
- 可见浏览器启动。
- 手动登录等待。
- 低频搜索结果采集。
- 可行时解析详情页。
- 基于规则的岗位评分。
- Excel 导出。

第一版不实现：

- 自动投递。
- 自动和招聘方沟通。
- 账号密码保存。
- 验证码绕过。
- 高并发采集。
- 云端部署。
