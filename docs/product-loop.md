# 产品闭环与路线图

## 当前闭环

```text
岗位收集 -> 技能点收集 -> 面试记录 -> 技能点补齐 -> 项目经验支撑 -> 继续筛选岗位
```

岗位收集阶段已经预留多来源：Boss 是第一期来源，后续可以继续接入猎聘、拉勾、智联、前程无忧或手动录入。无论来源如何，入库后都进入同一套评分、技能提炼、面试记录和补齐闭环。

### 1. 岗位收集

用户在 Boss 正常浏览岗位，只收藏自己感兴趣的岗位。系统保存岗位详情、职位描述、匹配分、跟进状态。

目标不是抓全量岗位，而是减少“看过就丢”的信息浪费。

### 2. 技能点收集

系统从职位描述中提炼技能点，并记录技能出现的上下文。技能库用于回答：

- 市场上反复要求什么技能？
- 哪些技能和高匹配岗位强相关？
- 哪些技能还缺项目支撑？

### 3. 面试记录

每次面试记录问题、结果、表现和复盘。问题关联技能点后，可以统计：

- 被问最多的技能。
- 答得不好的技能。
- 最近暴露短板的技能。

### 4. 技能点补齐

根据岗位和面试反馈更新技能库：

- 调整掌握程度。
- 补学习资料。
- 补项目证据。
- 为下次面试准备更具体的表达。

## 下一阶段建议

### P1：体验收口

- 面试记录详情页。
- 技能详情页展示“被哪些面试问过”。
- 岗位详情页直接展示相关技能缺口。
- 首页增加“本周待补技能”和“最近面试复盘”。

### P2：能力地图

- 技能点分层：语言、框架、数据库、中间件、架构、业务领域、软技能。
- 技能掌握程度和证据关联。
- 项目经验按技能点反查。

### P3：学习资料

- 给技能点维护学习链接。
- 后续可接搜索 API 或手动收藏资料。
- 根据薄弱技能生成学习清单。

### 暂缓

- 自动批量抓取 Boss 岗位详情。
- 自动投递。
- 账号登录自动化。

这些能力容易触发平台风控，也和当前“安全的个人求职工作台”定位冲突。
